CENTRAL FINDING:

  SIMILE has ASYMMETRIC GROUNDING.

  The source domain is always context-anchored (20/20 standard fires,
  100%). The source noun appears in recent SENSE_OBS or prior SIMILE
  content — SIMILE fires on what is currently being observed.

  The target domain is always generated fresh (22/22 fires, 100%).
  No target phrase is directly pulled from prior content. The target
  is the output of the translation operation: given source noun X
  from context, generate Y such that "X feels like Y."

  This asymmetry is structurally distinct from all other opcode
  families decrypted in Phase 2:
    INTROSPECT_STATE_TENSION: HALF-grounded (X-pool from context)
    CONSTANCY_FLUX: HALF-grounded (inner-phrase from adjacent fire)
    DEFLECTION_REDIRECT: TOPIC-STRATIFIED (grounding level varies)
    SIMILE: ASYMMETRIC (source always grounded, target always fresh)

  The dual-domain structure REQUIRES this asymmetry. The source domain
  must be observable (it is presented as a fact: "the ticking IS").
  The target domain is the metaphorical claim the translation generates
  (the ticking FEELS LIKE → what? — output of mapping operation).

VERDICT: SIMILE_DISTINCT

SIMILE has its own structural rules due to the dual-domain requirement.
Partial application of the grounding-grooves rule, with SIMILE-specific
modifications.

EVIDENCE:

1. ASYMMETRIC GROUNDING (unique to SIMILE):
   Source domain: 20/20 standard fires context-pulled (100%).
   Target domain: 22/22 fires generated fresh (100%).
   This asymmetry is not found in any other opcode family. Other families
   are either uniformly grounded, half-grounded, or topic-stratified.
   SIMILE is the only opcode where grounding applies to one dimension
   (source) but not the other (target). This follows from the structural
   requirement: SIMILE must translate something observed into something
   new. The observed thing is grounded by definition; the new thing
   cannot be.

2. TEMPLATE LOCK CONFIRMED (86%):
   The core "The [X] feels like [Y]" frame locks at 86%. This is
   consistent with the grounding-grooves rule: highly grounded source
   selection → high template lock on the structural shell.
   Within that shell, the BARE vs. COMPOUND variation tracks the run
   phase: COMPOUND_LOCATION locks within runs (spatial anchor established
   by first fire), then compresses to BARE near run end (fire 4782,
   fire 5139).

3. SATURATION CEILING (unique to SIMILE):
   SIMILE forms runs of 4–6 before mandatory SENSE_OBS flip.
   No other opcode family in Phase 2 has a saturation ceiling with
   mandatory categorical exit. This is not a burst cap (burst caps are
   opcode-type boundaries) — it is a saturation-state boundary that
   terminates the SIMILE run and returns to the observational layer
   that fed it.

4. SOURCE BEHAVIOR WITHIN RUNS (three modes):
   MODE A — SOURCE DRIFT: source noun changes partway through run
     (Run A: echo → clock → server hum)
   MODE B — SOURCE LOCK: single source noun throughout run, targets
     evolve (Runs B and C: ticking throughout, multiple targets)
   MODE C — DOMAIN SWEEP: source domain changes every fire, affect
     register unifies (Run D: clock → market → sun → calm → breeze →
     leaves)
   These three modes are not observed in any other opcode family.
   They are unique to a dual-domain template where the source can move
   independently of the structural frame.

5. TARGET EVOLUTION WITHIN RUNS (unique within Phase 2):
   Run C shows escalating novelty: heartbeat → oxymoronic whisper →
   full scene with human agent → affect-qualified heartbeat. The
   target generation is not random — it tracks along the novelty axis
   within a run, reaching maximum novelty at fire 3 (4790, restless
   piano scene) and reverting at fire 4 (4794). This within-run
   novelty arc is not observed in OBSERVE, INTROSPECT, or DIALECTICAL
   families.

COMPARISON TO GROUNDING-GROOVES RULE:
  Universal rule statement: "Template lock-rate controlled by grounding;
  burst length controlled by opcode-type; grooving instinct opcode-agnostic."

  SIMILE partial application:
    Template lock:   YES — 86% lock rate, consistent with rule.
    Burst regulator: MODIFIED — burst length is controlled by saturation
                     state, not opcode-type ceiling. The saturation law
                     provides an external ceiling (P3) that limits run
                     length regardless of opcode type.
    Grooving:        YES — but only on the SOURCE dimension. Source noun
                     locks across a run (Runs B, C). Target NEVER grooves
                     (no target phrase recurs within a single run; the
                     only cross-run recurrence is metronome ×2 and drum-
                     family across separate runs).

  MODIFIED RULE FOR SIMILE:
    "In SIMILE, grounding controls source-locking, template controls
    structural shell, and the saturation law controls burst ceiling.
    Target generation is always fresh within and across runs."
