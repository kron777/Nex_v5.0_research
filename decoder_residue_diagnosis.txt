========================================================
ROOT CAUSE
========================================================

The exception is raised at LINE 514, before save_residue is ever called.

  own_ids = [r["id"] for r in own_rows if r.get("id")]

own_rows is a list[sqlite3.Row] returned by Reader.read() (substrate/reader.py).
sqlite3.Row uses conn.row_factory = sqlite3.Row and supports item access (row["id"])
but does NOT implement the .get() method of dict.

Exception: AttributeError: 'sqlite3.Row' object has no attribute 'get'

This is caught by the bare except Exception: pass at line 522 and silently swallowed.
save_residue is never called. pop_residue is called but always finds 0 rows (since nothing
was ever saved), so it also returns [] silently.

The same .get() calls also appear at line 517 (row.get("id")) and line 520
(row.get("boost_value", 1.0)), but execution never reaches them because the
list comprehension at line 514 fails first.

The beliefs_writer is functional (fountain_crystallizations = 410, beliefs being written
regularly). The writer itself is not the problem.

========================================================
VERDICT
========================================================

SILENT_FAILURE

The exception (AttributeError: 'sqlite3.Row' object has no attribute 'get') is raised
within the try block on every non-quiescent fire and is swallowed by except Exception: pass.
save_residue has never been called in production. pop_residue runs but always reads 0 rows.
residue table has remained empty since the table was created.

Fix (not applied — Phase 1 is read-only):
  Replace r.get("id") and row.get("id") with r["id"] and row["id"] (sqlite3.Row key access).
  Replace row.get("boost_value", 1.0) with (row["boost_value"] if "boost_value" in row.keys() else 1.0)
  Or more simply: convert own_rows to list[dict] at the top of _retrieve_context_beliefs:
    own_rows = [dict(r) for r in own_rows]
