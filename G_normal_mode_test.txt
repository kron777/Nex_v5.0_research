  The "tick-feels-like-*" SIMILE pattern continued through all 8 post-switch
  fires without interruption. The prompt framing changed (focus_block removed)
  but the LLM output register did not change.

Finding 2: The groove suppressor preceded and survived the mode switch.
  Groove alerts for "feels like a" were firing before 12:29 and continued
  uninterrupted after. The mode switch did not reset or acknowledge the
  groove alert state.

Finding 3: Mode switch alone is insufficient to change output register.
  The register is controlled by: (a) prior context (SUSTAIN pattern),
  (b) groove lock. Neither is broken by a mode switch in isolation.
  The mode switch only changes the system prompt — it cannot change
  the LLM's prior context (which is drawn from the belief DB, which
  still contains the tick-groove beliefs) or reset the groove suppressor.

Finding 4: Crystallization is gated by groove suppressor.
  While the fountain is firing (8 events observed), 0 beliefs are being
  written. The groove suppressor is preventing crystallization of the
  tick-groove outputs. This is the first observed instance of the groove
  gate operating in real time.

  2 of 8 fires (events 1177, 1182) produced raw crypto JSON as output.
  droplet=None in both cases. This predates the mode switch and continues
  unchanged. The noise filter is not catching these before they reach the LLM.

  hot_branch=systems dominated before and after. The ai_research branch
  appeared once (event 1180) but returned to systems. Mode change does not

STEP 6: IMPLICATIONS FOR PROBE DESIGN
═══════════════════════════════════════════════════════════════════════

For the decryption framework (probe_runner.py), this experiment reveals
two additional confounds that must be controlled:

  A. Groove state: Before running a condition, the groove_alerts table must
     be checked. If a groove alert is active, the crystallizer will suppress
     outputs regardless of mode or prompt. Probe runs must wait for groove
     suppression to clear, or inject enough non-groove prior context
     to break the pattern before observing.

  B. Crystallization gate: fountain_events fires are NOT equivalent to
     new beliefs. The probe runner must track fountain_events, not just
     beliefs, to observe all outputs — including those blocked by the
     groove suppressor.

  C. Mode switch warm-up: Allow at least 3–5 fires after a mode switch
     before treating outputs as representative of the new mode. The
     LLM's effective prior context (from belief DB) lags the mode switch
     by several fire cycles.

  D. Noise events: Events with droplet=None are noise (raw JSON passed
     through). These must be excluded from template analysis.
