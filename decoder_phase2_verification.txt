Bug encountered during Phase 3 and immediately fixed:
  The original CREATE TABLE included FOREIGN KEY (belief_id) REFERENCES beliefs(id).
  beliefs lives in beliefs.db; fountain_retrieval_log is in dynamic.db. SQLite does
  not support cross-file foreign key enforcement. All INSERT attempts failed with:
    sqlite3.OperationalError: no such table: main.beliefs
  Fix: removed the cross-DB FK. fire_id -> fountain_events(id) retained (same DB).
  DROP TABLE IF EXISTS + CREATE TABLE in __init__ ensures automatic migration on boot.
  Fix committed to runtime repo (commit 7b021b1).

External feeds started via POST /api/sense/start after restart (GUI not available
in headless session). This is normal; Jon would toggle from GUI.
