========================================================
FRESHNESS WINDOW ANALYSIS
========================================================

The selector (world_bridge/selector.py) computes:
  freshness_window = max(60, cadence * CADENCE_MULTIPLIER)
  where cadence = mean inter-event gap over last CADENCE_LOOKBACK_SECONDS (3600s)
  and CADENCE_MULTIPLIER = 5

PROBLEM: Batch-fetching adapters insert N events (e.g. 20 arxiv papers, 28 BBC items) all with
the same integer timestamp (timestamp = int(time.time()) set ONCE before the loop in _helpers.py
parse_rss). This creates N events with identical timestamps within a burst, so inter-event gaps
within the burst are 0 seconds. The mean cadence becomes artificially small.

Example - arxiv_ai (3600s real poll interval, 20 events per burst):
  Between-burst max gap observed: 15,028s (4.2h)
  Mean cadence computed: ~92-178s (counting within-burst 0-gaps)
  Derived freshness window: max(60, 178*5) = 890s (14.8 min)
  Actual quiet period after burst: 3600s minus burst age
  Result: events expire from window after ~14.8 min; adapter won't poll for ~60 min

Example - bbc_news (900s poll, ~28-80 events per burst):
  Mean cadence computed: ~34s
  Derived freshness window: max(60, 34*5) = 170s (2.8 min)
  Actual poll gap: 900s
  Result: BBC events stale within 3 minutes; won't repoll for 15 minutes

Example - hacker_news (300s poll, 30 events per burst):
  Cadence computed: ~10s
  Derived freshness window: max(60, 10*5) = 60s (1.0 min)
  Actual poll gap: 300s (5 min)
  Result: HN events stale within 60 seconds; won't repoll for 5 minutes

========================================================
ROOT CAUSE
========================================================

Root cause: Freshness windows are derived from mean inter-event cadence, which is compressed
by within-burst 0-second gaps from batch-fetching adapters. This causes freshness windows to
be far shorter than actual poll intervals, creating long "stale" windows between batches.

Consequence: After each burst, external streams fall out of the freshness window within 60–890
seconds. The fountain fires every ~120s. During the gap between burst expiry and next poll, the
selector finds no active external streams and logs only internal.temporal + internal.meta_awareness.
