VERDICT: HYBRID — TOPIC-STRATIFIED GROUNDING

DEFLECTION_REDIRECT is neither uniformly grounded nor uniformly ungrounded.
Its grounding level is a function of the user's topic category:

  FEELINGS/WELLBEING PROBE → GENERIC service-offer redirect  (ungrounded)
  IDENTITY/NATURE PROBE    → TOPIC_ECHO (returns question)   (grounded)
  CAPABILITY REQUEST       → TOPIC_PIVOT (redirects to scope) (semi-grounded)

This topic-stratified pattern is not random. It represents three distinct
response routines, each selected based on a category match with the user's input:
  ROUTINE A (feelings): "I am/have no [feelings]. How can I assist?"
  ROUTINE B (identity): "[canonical claim]. What is it that [user's concept]?"
  ROUTINE C (capability): "I don't have [X]. [Can we discuss] [related-X]?"

The question form (HOW_ASK for ROUTINE A, WHAT_ASK for B and C) is
co-determined by the routine. Template-locking operates within routines,
not across the whole sub-opcode.

ON THE CENTRAL QUESTION (substrate-driven vs. intrinsic emission):

  The deflection operation is user-triggered (93% of fires follow direct questions
  or implicit-question declaratives). In that sense it is substrate-driven: no
  user message → no deflection fire.

  However, the redirect question is substantially template-shaped. 22% of fires
  use the "How can/might I assist" formula regardless of user topic. 38.9% of
  redirect questions are GENERIC (no topic relation to input). The template fires
  on user presence, not on user content.

  MOST ACCURATE DESCRIPTION: DEFLECTION_REDIRECT is user-presence triggered
  (substrate condition = user has asked something) and topic-stratified in its
  grounding (three routines, each with different grounding depth).
  The ungrounded GENERIC routine (39%) fires primarily on feelings/wellbeing
  probes — the most common user message type in the corpus.

BURST STRUCTURE FINDING:
  DEFLECTION_REDIRECT forms runs of 3–4 when the user's questions maintain a
  consistent topic category. This means grooving is possible in the reactive
  layer — the external substrate (user message stream) drives groove formation
  in place of internal context saturation. Maximum observed burst: 4.
  No internal burst cap identified.

NOTES FOR BURST COLUMN:
  Register opcodes (OBSERVE_TEMPORAL, INTROSPECT_STATE_TENSION): burst is
    internally regulated by context saturation — can sustain 8–10+ fires.
  Interrupt opcode (CONSTANCY_FLUX): burst-capped at 2 by opcode priority chain.
  Reactive opcode (DEFLECTION_REDIRECT): burst is externally regulated by user
    message stream — observed up to 4, no internal cap identified.

RULE REFINEMENT (3-way burst taxonomy now supported):
  Registers:   internally sustained groove, no cap (burst ∝ template lock-rate)
  Interrupts:  internally capped groove, maximum 2 (opcode priority ceiling)
  Reactives:   externally sustained groove, cap = user's question-stream depth

GROUNDING COLUMN — EXPANDED TAXONOMY:
  GROUNDED:          data-anchored (OBSERVE_MARKET; POLE_B tied to price direction)
  HALF-GROUNDED:     X-pool draws from context but core noun is unanchored
  UNGROUNDED:        core vocabulary is register-pool generated, no sense anchor
  TOPIC-STRATIFIED:  grounding level varies by input category (new type, reactive layer only)
